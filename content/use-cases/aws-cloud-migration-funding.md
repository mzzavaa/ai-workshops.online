## AWS Funds Cloud Migrations

Organizations migrating workloads to AWS or modernizing existing cloud infrastructure can access substantial funding through AWS programs. Depending on the scale and type of migration, funding can range from credit-based support to significant cash programs - up to EUR 400,000 for qualifying migrations.

This funding is available but underutilized. Many organizations either don't know it exists or find the program landscape confusing. This page explains what is available and how to access it.

## AWS Migration Funding Programs

### AWS Migration Acceleration Program (MAP)

MAP is the primary program for large-scale migrations. It provides funding across three phases:

**Assess:** Funding for migration readiness assessment, business case development, and portfolio analysis.

**Mobilize:** Funding for migration planning, skill building, and landing zone setup. This phase typically covers 25% of estimated migration costs.

**Migrate and Modernize:** The largest funding phase, covering actual migration execution. Qualifying projects can receive funding equivalent to a significant portion of their AWS spend during migration.

MAP is designed for migrations of meaningful scale - typically 50+ servers or equivalent workload volume.

### AWS Optimization and Licensing Assessment (OLA)

For organizations with significant on-premises Microsoft or Oracle licensing costs, OLA provides funding for an assessment that identifies opportunities to reduce licensing costs through cloud optimization. This is particularly valuable for organizations running SQL Server, Windows Server, or Oracle Database.

### Partner-Led Migration Funding

When working with an AWS Migration Competency Partner to deliver the migration, additional funding tiers become available. The partner relationship unlocks programs that are not accessible through direct customer applications.

This includes:
- Migration incentive funds
- Partner delivery credits
- Co-investment programs for strategic migrations

### Modernization Funding

Beyond lift-and-shift migration, AWS funds modernization projects: containerization, serverless refactoring, database migration to managed services, and application re-platforming. These projects often qualify for separate funding streams.

## How the Process Works

1. **Migration readiness assessment** - Understand current state, define migration scope, identify dependencies
2. **Business case development** - Calculate TCO, identify cost savings, quantify risk reduction
3. **Program identification** - Work with an AWS partner to identify which programs apply and in what combination
4. **Application and approval** - Submit through the appropriate channel (partner-led or direct)
5. **Delivery and milestone-based funding** - Funding is typically released against defined milestones

## What Makes a Strong Application

Strong applications share several characteristics:

- Clear scope definition with measurable migration milestones
- Documented business case with TCO analysis
- Committed internal team and executive sponsorship
- Partnership with an experienced AWS delivery partner
- Realistic timeline with defined go-live date
